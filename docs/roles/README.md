# BenchSight Role Guides

Copy-paste the appropriate guide at the start of each chat session to specialize the LLM for that task.

## Available Roles

| Role | File | Focus |
|------|------|-------|
| **Project Manager** | [PM.md](PM.md) | Coordination, status, planning |
| **ETL Engineer** | [ETL_ENGINEER.md](ETL_ENGINEER.md) | Build new tables, fix data issues |
| **ETL Maintenance** | [ETL_MAINTENANCE.md](ETL_MAINTENANCE.md) | Bug fixes, incremental updates |
| **Frontend Dev** | [FRONTEND.md](FRONTEND.md) | UI tracker, dashboard components |
| **Backend Dev** | [BACKEND.md](BACKEND.md) | API, Supabase, server logic |
| **Data Scientist** | [DATA_SCIENCE.md](DATA_SCIENCE.md) | Analytics, ML models, stats |
| **Dashboard/Viz** | [DASHBOARD.md](DASHBOARD.md) | Charts, reports, visualizations |
| **Code Review** | [CODE_REVIEW.md](CODE_REVIEW.md) | Review PRs, quality checks |
| **Computer Vision** | [CV.md](CV.md) | Video analysis, player tracking |
| **Tracker Dev** | [TRACKER.md](TRACKER.md) | Event tracking UI |

## Usage

1. Open new chat
2. Upload benchsight zip
3. Paste the appropriate role guide FIRST
4. Then describe your task

## Creating New Roles

Add a new markdown file following this template:

```markdown
# Role: [ROLE NAME]

## Scope
What this role does and doesn't do.

## First Steps
1. Read specific files
2. Check specific things
3. ...

## Rules
- Specific constraints
- Do/don't lists

## Common Tasks
- Task patterns for this role
```
