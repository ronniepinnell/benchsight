Overview

BenchSight uses Supabase as its database backend. Both the Dashboard and Portal connect directly to Supabase from the browser using the Supabase JavaScript client.

1. Get Supabase Credentials

Create a Supabase Project

  1. Go to supabase.com and sign up/login
  2. Click "New Project"
  3. Choose organization, name, password, region
  4. Wait for project to be created (~2 minutes)

Get API Credentials

  1. In your project, go to Settings â†’ API
  2. Copy the Project URL (looks like https://xxxxx.supabase.co)
  3. Copy the anon public key (starts with eyJ...)
âš ï¸ Security Note: The anon key is safe to use in browser apps. It only allows access based on Row Level Security (RLS) policies. For BenchSight, we use it for read-only dashboard access.

2. Configure Dashboard/Portal

Option A: Manual Configuration (Per Browser)

  1. Open the Tracker or Portal
  2. Click âš™ï¸ Settings
  3. Enter Supabase URL and Key
  4. Click "Save"

Settings are saved to localStorage and persist across sessions.

Option B: Pre-configured Deployment

For public deployments, you can hardcode credentials:

// In dashboard HTML, add before init():
const SUPABASE_URL = 'https://your-project.supabase.co';
const SUPABASE_KEY = 'eyJ...your-anon-key...';

// Then in init():
sb = supabase.createClient(SUPABASE_URL, SUPABASE_KEY);

3. Load Data into Supabase

Method 1: Supabase Sync Script (Recommended)

# From project directory
python scripts/supabase_sync.py --dims --facts

# Or specific tables
python scripts/supabase_sync.py --tables dim_player,dim_team,fact_events

Method 2: Manual CSV Import

  1. In Supabase, go to Table Editor
  2. Click "Create a new table" or select existing
  3. Click â‹® menu â†’ "Import data from CSV"
  4. Upload your CSV from data/output/

Method 3: SQL Editor

Use the SQL Editor in Supabase to run custom queries or create tables:

-- Example: Create fact_events if not exists
CREATE TABLE IF NOT EXISTS fact_events (
  event_id SERIAL PRIMARY KEY,
  game_id INT,
  event_index INT,
  period INT,
  event_type TEXT,
  event_detail TEXT,
  -- ... more columns
);

-- Import from local CSV (use Supabase CLI)

4. Deploy Dashboard

Option A: Static Hosting (Render, Netlify, Vercel)

  1. Create account on hosting service
  2. Create new "Static Site" project
  3. Upload or connect Git repo with ui/dashboard/ folder
  4. Set build command: (none needed)
  5. Set publish directory: ui/dashboard
  6. Deploy!

Option B: Wix (Custom Code Embed)

  1. In Wix Editor, add "Embed HTML" element
  2. Paste the entire dashboard HTML
  3. Or use iframe pointing to hosted version

Option C: GitHub Pages

  1. Push project to GitHub
  2. Go to Settings â†’ Pages
  3. Select source branch and folder
  4. Access at username.github.io/repo/ui/dashboard/

6. Troubleshooting

âŒ "Connection failed"

  • Check URL format (must include https://)
  • Verify key is the "anon public" key, not service role
  • Check browser console for CORS errors

âŒ "No data showing"

  • Verify tables exist in Supabase
  • Check table names match exactly (case-sensitive)
  • Run ETL and sync script first
  • Check RLS policies allow read access

âŒ CORS Errors

  • Supabase handles CORS automatically
  • If using custom domain, add to allowed origins
  • Check browser extensions aren't blocking
